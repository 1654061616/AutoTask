这些是 Windows GDI API 的标准常量，用于获取显示器的 DPI（每英寸像素数）：

| 常量 | 值 | 含义 |
|------|---|------|
| `LOGPIXELSX` | 88 | 水平方向的 DPI 值 |
| `LOGPIXELSY` | 90 | 垂直方向的 DPI 值 |
| `96.0` | - | Windows 默认 DPI（对应 100% 缩放） |

## 工作原理

```python
hdc = user32.GetDC(0)  # 获取屏幕设备上下文
dpi_x = ctypes.windll.gdi32.GetDeviceCaps(hdc, 88)  # 获取水平DPI
dpi_y = ctypes.windll.gdi32.GetDeviceCaps(hdc, 90)  # 获取垂直DPI
user32.ReleaseDC(0, hdc)

scale_x = dpi_x / 96.0  # 计算缩放比例
scale_y = dpi_y / 96.0
```

例如在 125% 缩放的显示器上：
- `dpi_x = 120`（96 × 1.25）
- `scale_x = 1.25`

这样就能将 Qt 的逻辑坐标（已除以缩放比例）转换为物理像素坐标（自动化执行时需要的坐标）。

```python
LOGPIXELSX = 88
LOGPIXELSY = 90
```

## 修改计划：为图片操作节点添加查找方向配置

### 需求分析
用户希望在查找图片时，可以指定查找的起始方向（从上到下、从左到右等），这样可以优先返回特定位置的匹配结果。

### 修改内容

#### 1. 修改 `operations/image_recognition.py`
- 在 `find_image` 方法中添加 `direction` 参数
- 修改模板匹配逻辑，根据方向遍历匹配结果并优先返回符合方向的位置
- 支持的方向：
  - `default`：默认（返回匹配度最高的）
  - `left_to_right`：从左到右
  - `right_to_left`：从右到左
  - `top_to_bottom`：从上到下
  - `bottom_to_top`：从下到上

#### 2. 修改 `core/engine.py`
- 在 `_execute_image_find`、`_execute_image_click`、`_execute_image_exists` 方法中，传递 `direction` 参数给 `image_recognition.find_image`

#### 3. 修改 `gui/step_panels/image_panel.py`

- 在 `get_config` 中返回方向配置
- 在 `set_config` 中恢复方向配置

### 关键实现思路

当前 `find_image` 使用 `cv2.minMaxLoc` 直接返回匹配度最高的位置。要实现方向优先查找：

1. 使用 `cv2.matchTemplate` 获取匹配结果矩阵
2. 根据方向遍历矩阵，找到第一个匹配度超过阈值的位置
3. 例如"从上到下"：按行从上到下遍历，每行按列从左到右遍历，返回第一个超过阈值的位置

### 影响范围
- **新增配置项**：`direction`（字符串类型）
- **向后兼容**：默认值为 `default`，与现有行为一致

## 修复方案

在 [gui/step_panels/__init__.py](file:///e:/A_PycharmProjects/AutoTask/gui/step_panels/__init__.py) 中新增 `_get_real_screen_pos()` 方法：

```python
def _get_real_screen_pos(self, global_pos):
    try:
        import ctypes
        user32 = ctypes.windll.user32
        
        hdc = user32.GetDC(0)
        dpi_x = ctypes.windll.gdi32.GetDeviceCaps(hdc, 88)  # LOGPIXELSX
        dpi_y = ctypes.windll.gdi32.GetDeviceCaps(hdc, 90)  # LOGPIXELSY
        user32.ReleaseDC(0, hdc)
        
        scale_x = dpi_x / 96.0
        scale_y = dpi_y / 96.0
        
        return QPoint(int(global_pos.x() * scale_x), int(global_pos.y() * scale_y))
    except:
        return global_pos
```

该方法通过 Windows API 获取实际 DPI，计算缩放比例（默认 96 DPI = 100%），然后将 Qt 逻辑坐标转换为实际像素坐标，确保与 `ImageGrab.grab()` 使用的坐标一致。
